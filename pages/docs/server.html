Server

Patties exposes two server entrypoints: createServer() for an in-process fetchable server handle, and startServer() for a real Bun.serve listener.

createServer

createServer is useful for tests or internal embedding. It compiles the route patterns and returns a fetch(req) function without binding a port.

startServer

startServer boots a real Bun server and forwards unmatched requests to the configured fallback handler.

Supported server options include:

  • port
  • hostname
  • unix
  • reusePort
  • routes
  • fallback
  • dev
  • devServer
  • staticRoutes

Dev-only routes

When dev: true, Patties injects internal routes for:

  • __patties_open to open files in an editor
  • __patties_hmr for hot-module reload websocket traffic

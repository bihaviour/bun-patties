# `patties` package conventions

This is the framework itself — the runtime, CLI, build pipeline, dev server,
renderer, router, middleware, and adapters. Most things you'd add to a
Bun-native full-stack framework live somewhere under `src/`.

Project-wide rules live in `../../CLAUDE.md`. Read those first. The notes
below are specific to working inside this package.

## Source layout

- `src/router/` — filesystem route scan + compile to Bun route patterns
- `src/render/` — React SSR (`renderToReadableStream`) + dev error overlay + island marker emission
- `src/client/` — browser hydration runtime; scans `[data-island]` markers
- `src/build/` — route + island scan, client/server entry generation, `Bun.build` pipeline. `adapter.bun.compile` (production) runs `bun build --compile` against the generated server-entry SOURCE and embeds app/public + client chunks via `with { type: "file" }` / `Bun.embeddedFiles`; `--compile` with `--target edge` is a config-time error
- `src/middleware/` — composition, `PattiesContext`. `request-id.ts` mints `ctx.requestId` (echoed as `X-Request-Id`, inbound junk ignored); the router finalizer folds in the cookie flush and dev-only `Server-Timing`
- `src/dev/` — `bun --hot` / `bun --watch` integration, HMR WebSocket, `setupDevClient` (bundles islands in dev and serves `/_patties/client/*`)
- `src/server/` — `startServer` / `createServer` around `Bun.serve`
- `src/config/` — `defineConfig`, env + secrets loading, Zod schemas. `config.agentsMd.path` (string or string[]) picks where the manifest lands; default is `CLAUDE.md`
- `src/ai/` — optional agents / tools / jobs primitives (Anthropic SDK is an optional peer dep)
- `src/agents-md/` — manifest generator + `writeManifestToFile` (splices into a fenced section so rules around it survive regeneration)
- `src/plugin/` — plugin system
- `src/adapters/` — `bun` and `edge` runtime adapters

## Public entry points

Declared in `package.json#exports`:

`patties`, `patties/router`, `patties/render`, `patties/client`,
`patties/build`, `patties/dev`, `patties/config`, `patties/middleware`,
`patties/server`, `patties/ai`, `patties/agents-md`, `patties/plugin`.

CLI at `bin/patties.ts` → `src/cli/index.ts` (`dev` / `build` / `run` / `deploy` / `secret` / `doctor` / `add` / `ui` / `view` / `update` / `upgrade` / `migrate`). `run` is the opt-in monorepo task orchestrator: `patties run <task>` wraps each workspace `(package, task)` unit with a content-addressed output cache + affected detection (`src/cli/run.ts` + `src/cli/run/`, framework spec 27). Its cache is 100% Bun builtins (`Bun.CryptoHasher` sha256 key, `Bun.Glob`, `Bun.Archive` tarballs, `bun:sqlite` WAL index, `Bun.$` git, `Bun.spawn`); per-task `inputs`/`outputs`/`env`/`cache` come from a `tasks` block in `patties.config.ts`. `patties add <component>` stamps shadcn-compatible UI source from the `patties-ui` catalog into `app/components/ui/` (it also resolves local paths, https URLs, and `@ns/<name>` namespaced registries); `ui init` scaffolds tokens + helpers, `ui build` emits a fetchable registry, `view`/`update` preview-then-stamp, and `migrate` codemods radix imports / RTL logical props. See `src/cli/commands/` and `.claude/rules/ui-catalog.md`. `upgrade` bumps the project's `patties` dependency to the newest release on its channel (stable→`latest`, prerelease→its pre tag); the CLI also prints a cached, channel-aware "update available" banner on startup (`src/cli/update-check.ts`, opt out with `--no-update-check` / `NO_UPDATE_NOTIFIER`). `doctor` aggregates read-only project-hygiene checks (toolchain probes, `bun audit` / `bun outdated`, lockfile sync, config + biome validity, and monorepo single-React / catalog invariants) into one aligned report with a remedy per finding and a CI exit code (`src/cli/doctor.ts` + `src/cli/doctor/`, cli spec 20); Phase 1 is report-only — no `--fix`. Width-aware output utilities live in `src/cli/ansi.ts` (`Bun.stringWidth` / `stripANSI` / `wrapAnsi`).

## Dev mode wiring

`patties dev` re-execs under `bun --hot --preserve-symlinks` and resolves the
re-exec entry through the user-project copy of the bin
(`<cwd>/node_modules/patties/bin/patties.ts` when present). This keeps module
resolution rooted in the app even when `patties` is linked from a workspace —
without it, two copies of React load and SSR hooks blow up.

Templates wire dev mode via `setupDevClient({ appDir })` from `patties/dev`:
it bundles islands in development, mutates a `ClientManifest` the renderer
holds by reference, and returns a `BunRoutes` map for `/_patties/client/*`.
Pages emit hydration markers only when wrapped in `<Island name=…>` — direct
island imports SSR fine but never hydrate.

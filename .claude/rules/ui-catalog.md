# UI catalog rule

`packages/patties-ui` is the shadcn-compatible, **copy-in** component catalog. `patties add <component>` stamps source into the user's `app/components/ui/` — components are never imported from a versioned npm package. All 60 components ship (`status: "completed"`).

- **Registry is the source of truth.** `src/schema.ts` (`ComponentEntrySchema`, zod) is the source of truth; `src/types.ts` re-derives via `z.infer`. `src/registry.ts` is what the CLI reads, and ships as the `patties-ui/schema` + `patties-ui/registry` exports. A template under `templates/` only becomes stampable once it has a `registry.ts` entry with `status: "completed"`. Edit the registry, not just the template dir.
- **SSR-first, island by opt-in.** Every component file exports `island: true | false | "subtree"` so the build can decide whether to ship it to the client. Static-by-default; `"subtree"` is a static shell inheriting a nested island. No top-level `window` / `document` / `process` access.
- **CLI surface.** `ui init` scaffolds the catalog (tokens + `_internal` helpers); `add` stamps (diff-aware, with `--theme neutral|slate|stone|zinc` presets from `templates/themes/`); `view` prints source before stamping; `update` re-stamps after showing the diff; `migrate` codemods radix imports + RTL logical properties; `ui build` emits a fetchable `registry.json` + per-component payloads for third-party authors.
- **`patties add` is CI-safe and Bun-native.** It refuses to run under `NODE_ENV=production`, and **never** runs `npm/bun install` — it edits `package.json` and tells the user to run `bun install`. Destination defaults to `app/components/ui/` (helpers in `_internal/`, tokens in `app/styles/tokens.css`) and is overridable via the optional `ui` config block (`componentsDir` / `internalDir` / `tokensFile`); there is still no `components.json` — config is convention. `add` also resolves local paths, https URLs, and `@ns/<name>` namespaced registries (`ui.registries` maps `@namespace` → https base or local path); external source is schema-validated, previewed, written verbatim, and never executed — http is rejected without `--allow-insecure`.
- **React 19 style.** No `forwardRef`; refs are plain props. Radix-class primitives via the unified `radix-ui` import. Theming is Tailwind v4 + CSS variables in `app/styles/tokens.css`, dark mode by `.dark` on `<html>` with no JS theming dep.
- **Specs** live in the sibling `bun-patties-docs` repo under `agent_specs/ui/archive/` (phases 0–4, all archived); `00-overview.md` is the catalog index. Parity against upstream shadcn is tracked in `agent_specs/research/patties-ui-parity-matrix.md`.

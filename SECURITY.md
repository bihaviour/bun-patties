# Security Policy

## Supported versions

Patties is pre-1.0 and under active development. Security fixes land on the
latest published release of each package (`patties`, `create-patties`,
`patties-ui`). Older versions are not patched — please upgrade to the latest
before reporting.

## Reporting a vulnerability

**Please do not report security issues through public GitHub issues, pull
requests, or discussions.**

Instead, report privately via either:

- **Email:** baita@bihaviour.com — include "SECURITY" in the subject line.
- **GitHub:** the repository's **Security → Report a vulnerability** tab
  (private advisories), if enabled.

Please include as much of the following as you can:

- The package and version affected (`bun --version` and the patties version).
- A description of the issue and its impact.
- Steps to reproduce, ideally a minimal proof of concept.
- Any suggested remediation.

## What to expect

- **Acknowledgement** within 72 hours.
- An initial assessment and severity triage shortly after.
- Progress updates as a fix is developed, and coordination on a disclosure
  timeline. We aim to ship a fix before any public disclosure.
- Credit for the reporter in the release notes, unless you prefer to remain
  anonymous.

We appreciate responsible disclosure and the time you take to make Patties
safer for everyone.
